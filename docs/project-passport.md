# Паспорт проекта

## Название проекта

Веб-сайт для онлайн заказов в БарберШоп

## Предметная область

Разрабатывается сайт барбершопа (мужской парикмахерской). Организация предоставляет услуги по мужским стрижкам, бритью, уходу за бородой и усами, а также продаёт сопутствующую косметику (воски, масла, гели для укладки). Сайт служит витриной и инструментом для онлайн-записи. 

## Проблема

В настоящее время клиентам неудобно записываться в барбершоп: звонки по телефону часто неудобны или не принимаются, социальные сети не дают полной информации о свободных слотах, а администратор тратит много времени на ручную сверку расписания барберов и обработку заявок из разных каналов (звонки, VK). Отсутствует единый актуальный каталог услуг с ценами и возможностью онлайн-бронирования.

## Цель проекта

Разработать веб-сайт барбершопа, позволяющий пользователям просматривать услуги и мастеров, записываться онлайн в несколько кликов, а администратору — управлять контентом и обрабатывать записи, автоматически видя занятость барберов. 

## Целевая аудитория

Новые клиенты, которые ищут барбершоп рядом с домом/работой, хотят ознакомиться с ценами, портфолио мастеров и записаться на первую стрижку. 

Постоянные клиенты, которым нужно регулярно записываться к своему мастеру, отслеживать историю посещений и получать напоминания. 

Администратор барбершопа, который управляет расписанием, подтверждает записи и контролирует наполнение сайта (услуги, мастера). 

## Роли пользователей

Гость — это неавторизованный посетитель сайта. Он имеет доступ только к публичной части: может просматривать главную страницу, каталог услуг, страницы барберов, читать отзывы и контакты. Основные возможности гостя — это регистрация нового аккаунта и авторизация (вход в систему). 

Пользователь — это авторизованный клиент барбершопа. Он может выполнять основное действие системы — создавать запись на услугу, выбирая мастера и время. Также пользователь имеет доступ к личному кабинету, где может просматривать свои активные и прошедшие записи, а также отменять запись (например, за сутки до визита). Дополнительно он может выходить из системы. 

Администратор — это сотрудник барбершопа, который управляет всем контентом и бизнес-процессами. Он может добавлять, редактировать и удалять услуги, а также управлять карточками барберов. Кроме того, администратор видит все записи всех пользователей, может подтверждать их, отмечать как выполненные или отменять по своей инициативе. В его задачи также входит управление расписанием (например, блокировка занятых слотов). 

## Контентная сущность

Контентной сущностью является «Услуга». Это основной наполняющий контент сайта, который описывает, что именно предлагает барбершоп (например, «Классическая мужская стрижка», «Бритьё опасной бритвой», «Коррекция бороды»). Каждая услуга имеет название, цену, длительность и описание. 

## Операционная сущность

Операционной сущностью является «Запись на услугу» (бронирование). Именно эта сущность фиксирует действие пользователя. Запись связывает между собой пользователя, выбранную услугу и конкретного барбера, а также содержит дату и время визита, статус (новая, подтверждена, выполнена, отменена) и, возможно, комментарий клиента. 

## Основные функции

Гость 

просмотр главной страницы; 

просмотр списка услуг; 

просмотр детальной страницы услуги; 

просмотр списка барберов; 

просмотр карточки барбера; 

регистрация; 

авторизация. 

Пользователь 

создание записи (выбор услуги -> выбор барбера -> выбор даты/времени -> подтверждение); 

просмотр своих записей («Мои записи»); 

отмена активной записи; 

выход из системы. 

Администратор 

добавление услуги (название, цена, длительность, описание); 

редактирование услуги; 

удаление услуги; 

добавление/редактирование/удаление барбера (имя, фото ,специализация); 

просмотр всех записей (с фильтром по дате, барберу); 

подтверждение записи; 

отметка записи как выполненной; 

отмена записи (с указанием причины для клиента);

## Данные, которые необходимо хранить

Пользователи (id, email, пароль, имя, телефон, роль: клиент/админ); 

Роли пользователей (id, название: admin, client); 

Услуги (id, название, цена, длительность в минутах, описание, изображение); 

Барберы (id, имя, фото, специализация, опыт работы, описание); 

Записи (id, id_пользователя, id_услуги, id_барбера, дата и время начала, статус, комментарий клиента); 

Статусы записей (id, название: новая, подтверждена, выполнена, отменена клиентом, отменена админом). 

## Планируемые страницы

Страницы проекта 

Публичная часть (доступ гостю / пользователю): 

Главная страница — приветствие, преимущества, популярные услуги, отзывы, призыв к записи. 

Каталог услуг — список всех услуг с ценой и длительностью. 

Детальная страница услуги — описание, цена, фото, кнопка «Записаться». 

Каталог барберов — карточки мастеров. 

Карточка барбера — фото, имя, опыт, специализация, список его записей (блоки). 

Страница регистрации — форма (имя, email, телефон, пароль). 

Страница авторизации — форма (email/телефон, пароль). 

Личный кабинет пользователя — отображение профиля (имя, контакты). 

Мои записи — список записей текущего пользователя с возможностью отмены. 

Форма основного действия (запись) — пошаговый выбор: услуга → барбер → дата/время → подтверждение. 

Административная панель (доступ по роли):  

Панель администратора (дашборд) — сводка по новым записям.  

Управление услугами — список, добавление, редактирование, удаление.  

Управление барберами — список, добавление, редактирование, удаление. 14. Управление записями (все записи) — таблица с фильтрами, возможность изменения статуса.
