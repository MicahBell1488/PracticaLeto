# Функциональные и нефункциональные требования

## Роли пользователей

### 1. Гость
Неавторизованный пользователь, который зашёл на сайт впервые или не хочет регистрироваться.

**Что может делать:**
- Смотреть главную страницу
- Смотреть список услуг и цены
- Смотреть портфолио работ
- Смотреть контакты и карту
- Читать отзывы
- Смотреть информацию о мастерах
- Зарегистрироваться
- Записаться на услугу (требуется ввод имени и телефона, регистрация не нужна)

### 2. Пользователь
Зарегистрированный и авторизованный клиент.

**Что может делать (всё, что гость, плюс):**
- Входить в личный кабинет
- Просматривать историю своих записей
- Отменять или переносить запись
- Оставлять отзывы после посещения
- Сохранять избранных мастеров
- Получать уведомления о напоминаниях (за день до записи)
- Редактировать свой профиль (имя, телефон, email)
- Смотреть историю своих посещений и потраченных сумм

### 3. Администратор
Владелец барбершопа или сотрудник с правами управления.

**Что может делать (всё, что пользователь, плюс):**
- Входить в админ-панель
- Управлять услугами (добавлять, редактировать, удалять, менять цену)
- Управлять мастерами (добавлять, удалять, назначать расписание)
- Управлять записями клиентов (видеть все записи, отменять, переносить)
- Подтверждать выполнение услуги
- Смотреть статистику (загрузка мастеров, доходы, популярные услуги)
- Управлять отзывами (удалять спам или оскорбления)
- Экспортировать отчёты (Excel/CSV)
- Добавлять и удалять администраторов

## Функциональные требования (по ролям)

### Для гостя:
| ID | Требование |
|----|-------------|
| FR-1 | На главной странице должен быть список услуг с ценами и временем |
| FR-2 | Должна быть форма записи с полями: имя, телефон, услуга, мастер, дата, время |
| FR-3 | После отправки формы гость должен получить уведомление "Запись отправлена" |
| FR-4 | Гость должен иметь возможность зарегистрироваться (имя, телефон, email, пароль) |
| FR-5 | Сайт должен корректно отображаться на телефонах и планшетах |
| FR-6 | Карта с расположением барбершопа должна быть интерактивной (кликабельной) |

### Для пользователя:
| ID | Требование |
|----|-------------|
| FR-7 | Пользователь должен входить по email/телефону и паролю |
| FR-8 | В личном кабинете должна быть история всех записей (дата, время, мастер, услуга, статус) |
| FR-9 | Пользователь должен отменять запись не менее чем за 2 часа до начала |
| FR-10 | Пользователь должен оставлять отзыв (оценка 1-5 звёзд + текст) |
| FR-11 | За день до записи пользователь должен получить напоминание по SMS или email |
| FR-12 | Пользователь может менять пароль и контактные данные |

### Для администратора:
| ID | Требование |
|----|-------------|
| FR-13 | Админ-панель должна быть отдельной страницей с паролем |
| FR-14 | Админ видит календарь со всеми записями всех мастеров |
| FR-15 | Админ может создавать нового мастера (имя, фото, описание, расписание) |
| FR-16 | Админ может задавать часы работы для каждого мастера отдельно |
| FR-17 | Админ видит отчёт по доходам за день/неделю/месяц |
| FR-18 | Админ может удалять любой отзыв |
| FR-19 | Админ может экспортировать список клиентов в Excel |

## Нефункциональные требования

| ID | Требование |
|----|-------------|
| NFR-1 | **Производительность:** Страницы должны загружаться быстрее 3 секунд |
| NFR-2 | **Доступность:** Сайт должен работать 99.5% времени |
| NFR-3 | **Безопасность:** Пароли должны храниться в зашифрованном виде |
| NFR-4 | **Безопасность:** Админ-панель должна быть защищена от несанкционированного доступа |
| NFR-5 | **Удобство:** Адаптивный дизайн для телефонов, планшетов, компьютеров |
| NFR-6 | **Удобство:** Количество кликов до записи не более 3 |
| NFR-7 | **Кроссбраузерность:** Сайт должен работать в Chrome, Firefox, Safari, Edge |
| NFR-8 | **Поддержка:** Код должен быть понятен другому разработчику |
| NFR-9 | **Масштабируемость:** Система должна выдерживать 100 одновременных пользователей |

## Ограничения проекта

| ID | Ограничение |
|----|-------------|
| C-1 | Стек технологий: HTML, CSS, JavaScript (фронтенд) |
| C-2 | Данные хранятся в localStorage или IndexedDB (нет сервера) |
| C-3 | Срок разработки: 2 недели |
| C-4 | Команда: 1 разработчик |
| C-5 | Бюджет: 0 рублей (бесплатные инструменты) |
| C-6 | Все данные теряются при очистке браузера (нет бэкенда) |
| C-7 | Регистрация и вход хранятся только в браузере пользователя |
